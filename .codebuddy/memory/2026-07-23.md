# 2026-07-23

## Ulysses Variant v2：原生 AG + 延迟 QKV 权重梯度 overlap

### 设计动机
v1 的 POST backward 使用 DeepGEMM 融合 AG+GEMM 算子，但 AG 本身的远端 payload 是 A2A 的 SP 倍（8×），AG kernel 占据 BWD 大部分时间（8K/SP8：AG kernel 475.7us vs 纯 GEMM 45.1us）。v2 改用原生 NCCL all_gather + 原生 GEMM，并将 QKV 权重梯度延迟到下一层 AG 通信窗口中 overlap。

### v2 实现
- **Forward**：与 v1 完全相同（GEMM+RS 融合）。
- **POST backward**：原生 `dist.all_gather_into_tensor`（comm stream）+ 原生 GEMM（default stream），不再使用 DeepGEMM AG+GEMM 融合算子。
- **QKV 权重梯度延迟**：`DeferredLinearFunction` 替换 Q/K/V 的 `nn.Linear.forward`，backward 立即计算 `grad_x` 但将 `(x, grad_output, weight)` 推入 `DeferredGradManager` 的队列。下一层 POST backward 的 AG 期间在 default stream 上执行这些延迟的权重梯度 GEMM。
- **最后一层**：`finalize_deferred_grads(model)` 在 `loss.backward()` 后显式刷新。
- **DDP 集成**：QKV 权重标记 `_deferred_grad = True`，DDP 排除这些参数；backward + finalize 后 `sync_deferred_grads` 手动 all-reduce。

### 代码结构
- `examples/wan21/autograd_ops_v2.py`：`DeferredGradManager`（comm stream + 队列 + AG buffer）、`DeferredLinearFunction`、`PostLinearV2Function`、`finalize_deferred_grads`、`sync_deferred_grads`。
- `examples/wan21/sp/variant_v2.py`：`FusedVariantV2Ulysses` 策略，继承 `UlyssesBase`，`_build_weights` 切 Wo + 替换 QKV。
- `examples/wan21/sp_training.py`：注册 `fused_var_v2` 策略。
- `examples/ulysses_variant_v2/`：test_correctness、bench_wan21_14b_train、bench_wan21_mem_train、bench_wan21_post_bwd、bench_variant_sweep、train_stability、plot 脚本、WAN21_ULYSSES_V2_BENCH.md。

### 关键正确性修复
- **dtype 不匹配**：grad_output 可能是 float32（来自 float32 norm/modulation），而 weight 是 bf16。在 `DeferredLinearFunction.backward`、`PostLinearV2Function.backward` 和 `DeferredGradManager.process` 中统一 cast 到 weight.dtype。
- **FA4 SM 10.3 兼容性**：FA4 4.0.0b14 的 `flash_fwd_sm100.py:162` 断言 `self.arch <= Arch.sm_110f`，但 `sm_110f` 被别名为 `sm_101f`（在枚举中位于 `sm_103` 之前），导致 B300（SM 10.3）断言失败。修复为 `Arch.sm_103f`。
- **Grad_W 4.44%**：与 v1 完全一致的 bf16 K 维度累积差异，非算子 bug。Loss curve 在 lr=1e-4 下 50 步 diff 0.0002%。

### 实验结果

#### POST backward 组件（8K/SP8, 100 iters）
- v1 融合 AG+GEMM：0.573ms；v2 原生 AG：0.213ms（NCCL 多 ring/多 SM 优化远优于 DeepGEMM 单 comm stream AG kernel）
- actual autograd：baseline 0.369ms，v1 0.765ms（2.075×），v2 0.393ms（1.065×）— **v2 仅为 v1 的 0.513×**
- 32K：v1 1.273ms，v2 0.974ms（0.765× v1）

#### 训练吞吐（40 层, 8K, 官方 14B 权重）
- **Manual sync**：serial 22,289 → v1 22,772 → **v2 23,142 tok/s**（+3.83% vs serial, +1.62% vs v1）。v2 BWD 142.31ms vs v1 148.00ms。
- **DDP**：serial 25,608 → v1 26,464 → v2 24,479 tok/s（-4.41% vs serial）。DDP 模式 v2 较慢因 QKV 手动 all-reduce 无法 overlap。

#### 吞吐扫描（manual sync, 2K-32K）
- v2 在**所有序列长度**下优于 v1（+0.9% ~ +4.7%）
- v2 在 2K-12K 范围内优于 serial（+1.1% ~ +5.1%）
- v2 BWD 在所有序列长度下快于 v1，8K 快 4.7%，12K 快 9.9%

#### 显存（40 层, 8K）
- serial 48,193MB → v1 38,280MB → v2 38,334MB（v2 多 54MB AG buffer，节省 20.5%）

#### 1000 步训练稳定性（DDP, 官方 14B 权重）
- serial final_loss=2.398, v2 final_loss=2.138（v2 更低）
- 前 900 步 diff < 0.02%，趋势完全一致
- v2 avg_tps=18,356 (0.969× serial)，peak 125.9GB（节省 6.2%）

### 环境修复
