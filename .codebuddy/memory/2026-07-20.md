# 2026-07-20

## 通用 GPU profiling SOP

- 权威操作文档改为 `docs/GPU_PROFILING_GUIDE.md`，适用于任意单卡/多卡算子、通信融合、autograd、DDP/FSDP 和完整训练；Ulysses POST backward 只作为附录案例。
- Profiling 采用五层法：正确性、组件、production Function/autograd、模块/子图、完整训练。Nsight Systems 只用于 kernel/copy/barrier/overlap 归因，不用其绝对时间作为最终吞吐。
- Ulysses 案例脚本为 `examples/ulysses_variant/bench_wan21_post_bwd.py`，内置 NVTX ranges，包括 `variant_AG+GEMM_(symm)`、baseline GEMM/A2A、actual baseline/variant POST BWD。
- Nsight 基本命令：`nsys profile --trace=cuda,nvtx,osrt,cublas --sample=none --cpuctxsw=none --wait=all ...`；统计使用 `nvtx_gpu_proj_sum`、`nvtx_kern_sum`、`cuda_gpu_kern_sum`、`cuda_gpu_mem_size_sum`、`cuda_gpu_trace`。
- 最重要经验：不要并行运行两个占满相同 8 张 GPU 的 benchmark；这会导致结果数倍波动。不同 shape/策略必须顺序跑，并重复至少三轮。
- 解释多 stream 时不能只看 default-stream CUDA Event；最终训练吞吐必须在前后 `torch.cuda.synchronize()` 后用 wall-clock，并对各 rank 取 MAX。
- `.nsys-rep` 和 `.sqlite` 是临时大文件，提取统计后删除，不提交到 git。

## 已确认数据

- Nsight 8K/SP8：AG kernel 约 475.7us；同形状纯 dX GEMM 45.1us；baseline dX/dW GEMM 42.2/32.9us；NCCL A2A kernel 46.9us；两个 symmetric barrier 合计约 51us。
- 无 profiler 重复测试：8K AG+GEMM 0.540–0.577ms；32K 0.982–1.030ms。
- 真实 production autograd POST BWD 三轮均值：8K variant 约为 baseline 1.34×；32K 约 1.97×。
- 结论：慢点是 AG 8×远端 payload、peer/chunk 调度和 kernel 内等待，不是 torch GEMM 本身。
