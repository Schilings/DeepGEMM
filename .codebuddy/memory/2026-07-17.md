## 迁移 GemmRSFunction 到 UnifiedSymmBuffer（单 buffer 服务 fwd+bwd、跨层共享）

- 用户回忆并确认：要 `GemmRSFunction`（`examples/wan21/autograd_ops.py:128`）支持「全局所有层、一个固定大小 buffer、fwd 和 bwd 都复用、不增加多余开销和显存」。
- 核对记忆：此设计**已实现**为 `deep_gemm/unified_buffer/UnifiedSymmBuffer`（`get_unified_symm_buffer`，2026-07-02 新增），被 `tests/comm/test_unified_buffer.py` 验证（2/8 卡 GEMM-RS + AG-GEMM 7/7 PASS）。`fused_variant.py:46-50` 那条「cannot share」旧注释对应的是**未迁移前的旧两独立 buffer 实现**。
- 完成迁移：
  - `fused_variant.py._create_buffers`：两独立 buffer（`GemmRSSymmBuffer`+`BF16AGGemmSymmBuffer`）→ 单个 `self.sym_post = get_unified_symm_buffer(group, bs, seq, dim)`（num_max_tokens_per_rank = bs*(seq//sp) == self.local_m，正好等于 gemm_rs 调用传的 local_m）。
  - `GemmRSFunction.forward` 用该 buffer 跑 GEMM+RS；`backward` 用 `sym_buffer.ag_x` 写 grad_y、`bf16_ag_gemm_nt(sym_buffer, ...)` 跑 AG+GEMM、从 `sym_buffer.ag_slots_x` 读 gathered grad_y 算权重梯度。
  - `share_buffers_from` 只共享 `sym_post`；`destroy_buffers` 只 destroy 一次。
- 正确性论证：`unified_buffer` 在**同一块物理内存**上给 fwd(RS 视图 partial@32) 和 bwd(AG 视图 ag_x@32 / ag_slots_x@32+sp*M*H*2) 各自 reinterpret；fwd/bwd 不并发，无交叉读。迁移模式与已验证的 `test_unified_buffer.py` 完全一致（ag_x 写 + bf16_ag_gemm_nt 传 unified buffer），故构造上正确。

## POST-only Ulysses 显存消融与 Function 重构

- 重建严格两臂消融：`serial` 全流程为 torch Q/K/V + 同步 NCCL A2A + torch SDPA + A2A/Wo；`fused_var` 只替换 POST 为 GEMM+RS，backward 用 AG+GEMM，其余代码完全共用。
- 删除冗余 `Wqkv/Wqkv_t/Wo_t` 参数；variant 切出 `Wo_r_local[dim, dim/SP]` 后注销完整 `model.o.weight`。FSDP2 改为逐层、`reshard_after_forward=True`，只忽略天然分片的 `Wo_r_local`；Adam 按 DTensor 本地 shard 分配。
- `autograd_ops.py` 收敛为 `FusedPostLinearFunction`；workspace 由策略 owner 持有并跨层借用，Function 不分配/销毁/清零 buffer。新增 `bf16_ag_gemm_nt_with_input` 隐藏 `.ag_x/.ag_slots_x` 并复用 gathered view 算 dW。
- 纠正旧判断：不能在每次 Function 中 `buffer.zero_()`。GEMM-RS barrier 自复位、AG C++ launcher 自行清 `slot_state`；真正竞态是 AG comm stream 在远端 `local_x` 尚未发布时提前 pull。当前高层入口在 staging 后做设备同步 + group barrier + 设备同步，2 卡连续 4/4 `grad_X rel=0`，fwd rel≈0.002964–0.002968。后续应将此发布协议下沉为纯 GPU/NVLink barrier 以移除 host sync。
- 重要纠正：上一轮把同一个 SP=8 group 又当作 FSDP group，导致 baseline Wo 也被切成 1/8，人为抹掉 variant 的核心收益。正确配置为本机 8 卡全做 SP、DP=1（若有 FSDP 应沿独立 DP 维），两者 attention 均恢复 FA4。B300×8、40 attention 层、BF16 weight/grad + FP32 Adam 实测：8K serial 48,191.3MB，variant 38,277.4MB（省 9,913.9MB/20.6%，workspace 160MB）；32K serial 68,686.5MB，variant 59,056.6MB（省 9,629.9MB/14.0%，workspace 640MB）。每层完整 Wo=50MB，variant 每卡省 7/8，按 weight+grad+Adam(m/v fp32)=6 倍计，40 层理论省约 10.5GB，足以覆盖 unified workspace。
