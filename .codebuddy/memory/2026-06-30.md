## Wan2.1 完整注意力 benchmark 重构（分层架构）
- 用户要求：解耦模型/SP策略/FSDP2/benchmark，面向训练框架集成，支持 FSDP2 权重梯度同步，只跑 THD（PackedSequence 省 GPU 显存）。
- 新建 benchmarks/wan21/ 包，分层架构：
  - config.py: Wan21Config（模型配置 dataclass）+ SPConfig（SP策略）+ TrainConfig（训练/FSDP2配置）
  - model.py: WanSelfAttention(nn.Module)，纯模型层，与并行策略无关。接收 device 参数。
    子方法：qkv_proj/apply_rope/attention/wo_proj，SP策略可拦截插入通信。
  - rope.py: 3D RoPE（Wan2.1 原生 T/H/W 三轴），rope_apply + rope_inverse
  - norm.py: WanRMSNorm（bf16 scale，forward 保持 bf16）
  - sp/base.py: UlyssesSPBase 抽象基类，定义 forward/backward 接口
    - PRE: GEMM+A2A-transpose（fused 或 torch）
    - ATTN: QK norm + RoPE + FA4
    - POST: 策略相关（子类实现）
    - backward 用 all-gather 方法：all-gather X_local + grad_y → 各 rank 有完整数据 → autograd 计算梯度
      （A2A-inverse 手动实现太难调对，all-gather 最简单且正确）
  - sp/standard.py: 标准 Ulysses（A2A-transpose + Wo GEMM POST）
  - sp/variant.py: Ulysses 变体（Wo row-split + GEMM+RS POST），backward 独立实现处理 N-sharded 输出
  - sp/torch_baseline.py: torch 原生基线（matmul + NCCL A2A）
  - fsdp2_utils.py: FSDP2 权重梯度同步（auto fully_shard 或 manual all-reduce 兜底）
  - bench_utils.py: 计时 + 正确性验证 + 分布式梯度验证
- benchmark 脚本 bench_wan21_ulysses_attn.py：统一入口，--variant 切换变体，--verify 小 shape 正确性验证
- verify_wan21_attn.py：2 卡快速正确性验证脚本（小 config，含 FWD+BWD 梯度 + 分布式梯度验证）
  关键修复：grad_y 必须在所有 rank 间 broadcast（不同 shape 的 randn 不会匹配！）
- 关键 bug 修复：
  1. WanRMSNorm scale 必须 bf16（默认 float32 导致输出 float32，FA4 要求 bf16）
  2. rope_apply 必须保持输入 dtype（.to(q.dtype) 在最后）
  3. nn.Linear 必须 dtype=bfloat16 创建
  4. torch.cuda.set_device(rank) 必须在 init_process_group 前调用
  5. backward 用 all-gather 方法避免 A2A-inverse 的 permute 陷阱
  6. variant backward 需要处理 N-sharded 输出（grad_y 是 [lm, local_N] 不是 [lm, dim]）
- 8 卡 THD benchmark 结果（iters=10, us=fused/torch）：
  STANDARD: 8K FWD 5538/1081 BWD 11095/12284 | 480p FWD 3442/3733 BWD 84334/88086 | 720p FWD 12561/13365 BWD 371004/385316
  VARIANT:  8K FWD 1169/1103 BWD 11350/12393 | 480p FWD 3209/3751 BWD 84433/88792 | 720p FWD 12118/13342 BWD 370721/384597
  注：FWD fused 在 8K 比 torch 慢（5538 vs 1081）是因为 fused op 有固定开销，小 shape 不划算；
      但 480p/720p fused FWD 比 torch 快（3442 vs 3733, 12561 vs 13365）。
  BWD 两者接近（all-gather 方法开销大，主要用于正确性验证，实际训练应用 A2A-inverse 优化）。
- 正确性验证（2卡 dim=5120 seq=8192）：FWD rel=0.0018, BWD grad_X rel=0.0016, grad_Wqkv rel=0.0000 → PASS
